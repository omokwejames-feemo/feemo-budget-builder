Chapter 01

Introduction & Overview

Feemo Budget Manager is a desktop application built for Nigerian film and TV productions. It replaces spreadsheet-based budgeting with a structured, auditable workflow — from first assumptions through to signed payment schedules and live spend tracking.

What Feemo does

  • Structures your production budget across 27 standard departments
  • Generates payment schedules with VAT and WHT calculations
  • Tracks actual spend against budget in real time
  • Projects monthly cashflow against installment receipts
  • Exports audit-ready PDFs and Excel reports

Interface overview

The application is divided into three zones:

  • Sidebar — navigation between screens, grouped into Overview, Finance, Payments and Project sections. The active project name and budget appear in the footer chip.
  • Topbar — live status pills showing project name, budget health, live spend percentage, and shoot day count. The theme toggle (☀️ / 🌙) and Upload Budget button live here.
  • Content area — the active screen fills this space. Each screen auto-saves its state.
ℹ️

Beta access: Feemo is currently available to authorised beta testers only. Your access key is issued by Feemovision Limited. Keys are valid for a fixed period — the app will notify you when your access nears expiry.

Chapter 02

Getting Started

From first launch to a fully loaded project in three steps.

Beta Gate

On first launch, Feemo presents the Beta Access screen. Enter your access key (format: FEEMO-XXXX-XXXX) and your email address, then tap Confirm. A one-time verification code will be sent to your email. Once verified, your session is saved and you won't be asked again until the key expires.

Home Screen — starting a project

After passing the gate, the Home Screen presents three entry points:

  • New Project — opens a blank project on the Assumptions screen.
  • Open Project — opens a .feemo file you previously saved, or picks from a recent projects list.
  • Upload Budget — parses an existing Excel production budget and populates the app automatically (see Chapter 10).
💡

Quick reopen: Recently opened projects are listed on the Home Screen and in the Open Project dialog. Double-clicking a .feemo file in Finder also opens it directly.

Saving your work

Press ⌘ S (Mac) or Ctrl S (Windows) at any time to save. Use File → Save As to export a named .feemo file to a location of your choice. An unsaved-changes indicator (●) appears next to the File nav item whenever there are uncommitted edits. Feemo also writes a crash-recovery snapshot before the window closes — if a session ends unexpectedly, you'll be offered a restore on next launch.

Light and Dark mode

Toggle between dark and light themes at any time using the ☀️ / 🌙 button in the top-right of the topbar. Your preference is saved across sessions.

Chapter 04

Assumptions

The foundation of every budget. All figures elsewhere in the app derive from the values you set here.

Project Details

Fill in your project identity and core parameters:

  • Title & Company — appear on every exported document.
  • Format — Feature Film, Series, Web Series, or Telenovela. Controls how shoot duration is calculated.
  • Total Budget — the approved production budget in your chosen currency.
  • Shoot Days — total number of principal photography days.
  • Currency — NGN, USD, GBP, or EUR. All values display in the selected currency.
  • Production Fee % — Department II (Contingency / Production Fee) is automatically calculated as this percentage of total budget and is read-only in the Budget screen.

Timeline

Set the duration in months for each production phase: Development, Pre-Production, Principal Photography, and Post-Production. The total feeds the Cashflow Projection chart and the current phase badge on the Shoot Days KPI card.

⚠️

If no start date or timeline is set, the Cashflow Projection cannot render and the Installment Advisor cannot suggest tranche timing. Set these values before using those features.

Funding Installments

Define the funding tranches for your production. Each installment has a label, a percentage of total budget, and a trigger month (relative to the project start date). The Installment Advisor (on the Payment Schedules screen) can suggest optimal timing automatically based on your cashflow projection.

  • Percentages must sum to 100% — an amber warning appears if they don't.
  • Reset to empty using the Reset Installments button.

Department Allocations

Distribute your total budget across the 27 standard departments as percentages. Each row shows both the percentage field and the derived currency amount — editing either updates the other in real time.

  • Percentages should total 100%. A running total is shown; an amber warning fires if it doesn't balance.
  • Department II (Contingency / Production Fee) is auto-calculated from your Production Fee % and is locked.
  • Use Reset Allocations to clear all percentages.
💡

Auto-detect from Excel: When you upload a budget file, Feemo detects department percentages from the workbook and offers to apply them automatically — with a Scale to 100% option if the detected total doesn't balance.

Chapter 05

Production Budget

The detailed line-item budget across all departments. Each line drives the Production Forecast cashflow.

Line items

Each budget line has the following fields:

  • Dept — the department code (A–II).
  • Schedule No. — budget schedule reference (e.g. G-01).
  • Detail — description of the cost item.
  • No. — number of people or units.
  • Qty — duration or multiplier (weeks, months, days).
  • Rate — unit cost.
  • Unit — how the rate is measured (e.g. per week, per day, flat).
  • I/E — marks whether the item is Included or Excluded from the main budget total.

The total for each line is No. × Qty × Rate. Department totals roll up into the allocation comparison.

Budget integrity check

Feemo runs a background integrity check every time line items, forecast overrides, or salary roles change. If the sum of line items differs from the Production Forecast total by more than 1%, a Mismatch Banner appears across Budget, Salary, Forecast, and Production screens. The Resolve Mismatch dialog offers three options:

  • Option A — proportionally scale the forecast to match the budget.
  • Option B — adjust the largest department's line items to absorb the gap.
  • Option C — dismiss and fix manually.
Chapter 06

Salary Forecast

Per-person monthly salary planning for your crew and cast.

Salary roles grid

Each role has:

  • Role — job title (e.g. Director of Photography).
  • Dept — the department this salary draws from.
  • Phase — the production phase(s) the role is active: Development, Pre-production, Shoot, Post, or All.
  • Monthly amounts — a column per production month. Enter the monthly salary for that role.

Over-allocation warning

The Salary Forecast compares the total salaries entered against the combined allocation for crew and cast departments (C, D, E, F). If salaries exceed the allocation a red warning banner shows the exact overage and the Grand Total turns red.

⚠️

Salary data is for planning visibility only — it no longer feeds the Production Forecast cashflow (which derives purely from line items ÷ active phase months since v2.2.0).

Chapter 07

Production Forecast

A monthly cashflow view derived automatically from your line items and installment schedule.

How the forecast is calculated

For each department, the forecast spend per month equals:

Monthly Spend = Department Line Items Total ÷ Number of Active Phase Months

Spend is distributed evenly across all active months, with the last month absorbing any rounding difference. This means updating a line item on the Budget screen instantly updates the forecast — there is no manual sync step.

Manual overrides

Individual cells can be overridden by typing directly into the forecast grid. Overridden cells are shown with an indicator. If overrides from a previous session are causing unexpected totals, use the Reset to Calculated Values button (visible whenever any overrides are active) to clear them and return to the formula-derived values.

Deficit detection

Months where cumulative spend exceeds cumulative income are flagged as deficit months. A Deficit pill appears in the topbar and the issues detector highlights affected months. The Installment Advisor on the Payment Schedules screen can resolve deficits automatically.

Chapter 08

Payment Schedules

Create, export, and track supplier payment schedules — with VAT and WHT applied automatically.

Creating a schedule

  1. Click New Schedule. A new schedule is created with the next sequential number (PS-001, PS-002, …).
  2. Set global VAT % and WHT % for the schedule, or override per row.
  3. Add payee rows: payee name, description, budget code (dept), bank details, and payment value.
  4. The payable amount per row is calculated as:
    Payment Value + (Value × VAT%) − (Value × WHT%)
  5. Fill in Prepared By, Reviewed By, Approved By fields.
  6. Click Export PDF to generate the schedule document for signing.

Installment Advisor

The collapsible Installment Advisor panel on this screen reads your forecast cashflow and calculates optimal installment timing. It:

  • Identifies deficit months where spend outpaces income.
  • Proposes 1–5 installment tranches with suggested months and percentages.
  • Renders an SVG cashflow chart showing spend, income, deficit shading, and tranche markers.
  • Writes the suggested installments directly to your Assumptions store when you click Apply.

Schedule statuses

Draft — created but not yet exported Exported — PDF generated, awaiting signature Approved — signed and recorded in Expenditure
⚠️

Draft schedules older than 48 hours appear as alerts on the Production Dashboard under the Alerts panel. Sign or archive them to clear the alert.

Chapter 09

Expenditure Tracker

The single source of truth for actual spend. When a payment schedule is signed and approved here, the amount deducts from the relevant department's budget and updates the Production Dashboard live.

Approving a schedule

  1. Select the payment schedule from the list.
  2. Confirm the signed PDF has been received.
  3. Set the approval date and click Approve & Record.
  4. The schedule moves to Approved status and the spend amount is added to the department's total in expenditureDeductions.

Once recorded, the Production Dashboard KPI cards, department list, and Recent Payments panel all update immediately.

What counts as "spent"

Only approved payment schedules contribute to Total Spent on the dashboard. Exported (unsigned) schedules are counted as pending and shown in the Alerts panel. This two-stage gate (Export → Approve) prevents provisional schedules from inflating your spend figures.

Chapter 10

Uploading a Budget from Excel

Import an existing Excel production budget and populate the app in one step — no re-keying required.

The 5-stage upload flow

1

2
Page Declaration
3
Parsing
4
Review
5
Confirm
  1. File Selection — drag and drop or browse to your .xlsx or .xls file. Files over 500 KB are processed off the main thread to prevent freezes.
  2. Page Declaration — tick which sheet types your file contains: Full Budget, Forecast, Salary Forecast, Dept Summary.
  3. Parsing — Feemo scans each sheet for column headers and classifies the document type. An animated progress bar tracks parsing.
  4. Review — a pre-population summary shows what was detected: total budget, line item count, salary role count, dept allocations, and payment schedule count. Green badges = will be set. Grey = not detected.
  5. Confirm — click Load into Project. If existing data is present, an overwrite warning lists what will be replaced.

Department allocation detection

If department figures are detected in the workbook, a dedicated dialog appears before the Review step. It lists each department, its detected percentage, and the grand total. If the total is not 100%, an amber warning appears with a Scale to 100% button that proportionally adjusts all percentages. Choose Apply Detected Allocations or Enter Manually to set them yourself on the Departments tab.

Upload audit log

After every successful upload a collapsible Upload Summary panel appears on the Production Budget screen showing the file name, timestamp, document type, and a full field-by-field breakdown of what was populated vs missing.

ℹ️

The parser is column-position-independent — it finds headers by name rather than fixed column positions, so Nigerian template formats with non-standard layouts are handled correctly.

Chapter 11

Export & Reports

Download a full Excel workbook or a one-page PDF spend summary — with live data from your project.

Live Spend Summary (top of screen)

When your project has a budget set, four KPI cards appear at the top of the Export screen showing real-time figures pulled from your approved expenditure:

  • Total Budget — the grand total entered on the Assumptions screen.
  • Total Spent — the sum of all approved payment schedules.
  • Remaining — budget minus spent; turns red if over budget.
  • Budget Used % — percentage consumed, colour-coded green / amber / red.

Download Excel Workbook

Exports the full production budget as a multi-sheet .xlsx file — one sheet per department, a summary sheet, and (when spend data exists) an Actuals tab showing approved deductions per department. On macOS the OS save dialog opens so you can choose the destination folder.

Download PDF Summary

Opens a new browser window with a print-ready single-page spend report. The page is generated entirely from live project data and includes:

  • Four KPI cards (Total Budget, Total Spent, Remaining, Budget Used %)
  • Department Breakdown table — budgeted vs spent per department, with status chips
  • Payment History table — every approved schedule sorted newest-first

Once the page opens, use ⌘ P (Mac) or Ctrl P (Windows) and choose Save as PDF in the print dialog. The page auto-triggers the print dialog on load.

💡

If no spend data exists yet, the "Download PDF Summary" button is disabled. Record approved payment schedules on the Expenditure screen first.

Department Spend Breakdown

Below the download buttons, a table lists every active department with a horizontal progress bar, spend vs budget values, and a colour-coded status chip: ON TRACK (more than 15% remaining), AT RISK (less than 15% remaining), or OVER BUDGET (spent exceeds allocation).

Payment History

Every approved expenditure schedule is listed here, sorted newest-first, with the schedule number, department, signed date, and amount. The footer row shows the total amount paid out across all schedules.

Pre-flight Checks

A checklist at the bottom of the Export screen shows the readiness of your project for export. Green ticks confirm required fields; grey items indicate missing data. All items should be green before sharing your workbook externally.
